# D-001 — Dockerfile: unpinned base image and root user

## Root Cause
`FROM python:latest` is used instead of a pinned version, and no `USER` directive exists so the container runs as root (UID 0).

## Files to Inspect
- `docker/Dockerfile`

## Exact Fix
```dockerfile
# Change FROM line
FROM python:3.11-slim

# Add before CMD
RUN addgroup --system appgroup && adduser --system --ingroup appgroup appuser
USER appuser
```

## Validation
```bash
./interview validate
```
Passes when: tag contains `major.minor` (e.g. `3.11`), a `USER` directive exists, and the user is not root.

## Common Wrong Fixes
- Using `python:3` — single major tag still changes with minor releases; validator rejects it
- Adding `USER root` — makes it worse
- Removing the FROM line — breaks the build

## Follow-Up Questions
- Why is `latest` dangerous in production? (Breaks reproducibility; image changes under you silently)
- What is the blast radius if the container runs as root and gets exploited?
- How would you enforce this policy across all team Dockerfiles? (Linting, CI gate, OPA admission)
