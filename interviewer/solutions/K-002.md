# K-002 — Service selector doesn't match deployment labels

## Root Cause
`k8s/service.yaml` has `selector: app: api-service` but the deployment's pod template has `labels: app: interview-api`. The Service has no matching endpoints so all requests return 502.

## Files to Inspect
- `k8s/service.yaml` — spec.selector
- `k8s/deployment.yaml` — spec.template.metadata.labels
- `kubectl get endpoints interview-api-svc` — will show `<none>`

## Exact Fix
```yaml
# k8s/service.yaml — change selector to match deployment labels
spec:
  selector:
    app: interview-api   # was: app: api-service
```

## Validation
```bash
./interview validate
```
Passes when service selector matches pod template labels.

## Common Wrong Fixes
- Restarting pods — doesn't fix the selector
- Changing the deployment label to match the wrong selector — works but names become inconsistent
- Creating a new Service instead of fixing the existing one

## Follow-Up Questions
- What command shows you which pods a Service is routing to? (`kubectl get endpoints`)
- How would you catch a selector mismatch in CI? (kubeval, conftest, helm lint)
- What is the difference between `spec.selector` on a Deployment and on a Service?
